Motivation

What motivates someone to learn a foreign language?

  • They want to travel to a foreign country and speak some of the local language.
  • You want to do business with people from a foreign country.
  • They have an uncle from Italy who knows ten words in English, but much prefers Italian, and they want to talk with him.
  • They want to flirt with that cute girl who doesn't speak English.

and many other countless reasons

People also have motivations to learn programming languages:

  • They want to build a script to automate some of their work
  • They want to build a website or app
  • They have an assignment due

The point is that just like people learn foreign languages in order to help them accomplish some goal, people learn programming languages in order to help them accomplish some goal. Of course there are exceptions where people learn just to learn. Most people who learn Latin aren't doing it to speak Latin with their neighbors. Similarly, if you interact with people who speak Spanish, learning Spanish would benefit you more than learning Greek.

Example Code Sucks

Most programming examples online suck. Let's say that Bob is trying to learn how to code. He might go online and read a tutorial which looks like this:

"Abstract classes can have method signatures which are defined in their subclasses. For example, the abstract class Animal can define the abstract method 'eat' which is defined in the Bear class and Fish class."

public abstract class Animal {
  public abstract void eat();
}

public class Bear extends Animal() {
  public void eat(){
    System.out.println("I'm a bear and I eat bamboo");
  }
}

public class Fish extends Animal() {
  public void eat(){
    System.out.println("I'm a fish and I eat algae");
  }
}

Bob has no idea how to code, and this example doesn't seem to help. How can subclassing the animal class help him script his data entry task in Excel?

It doesn't help. He's more confused than when he started.

There seems to be an unwritten rule that code samples can't solve problems. They have to be so abstract that they could never be pulled from a real code base. I'm not sure why this rule is followed so closely. [1]

Programming lectures, books, talks, videos, tutorials and courses. Some people find these to be very helpful. I don't. I can pass an entire programming course on Coursera, Udacity or at University without retaining much of anything which I learned.

In order for me to learn, I need to have a real, practical problem that I'm solving.

I suspect that many programmers are the same way. If I'm listening to a lecture about some new cool technology, I constantly ask myself "how can I use this to solve a problem I have?". If it can't solve one of my problems, I can't be bothered. I'll zone out. My brain won't bother to commit any of it to memory.
